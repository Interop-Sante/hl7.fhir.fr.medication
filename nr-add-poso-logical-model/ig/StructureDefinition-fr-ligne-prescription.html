Logical Model: Ligne de precription

Official URL: https://hl7.fr/ig/fhir/medication/StructureDefinition/fr-ligne-prescription Version: 0.1.0
Draft as of 2025-07-15 Computable Name: LignePrescription

Structuration d’une ligne de prescription en fonction des critères de la HAS.

Usages:

  • This Logical Model is not used by any profiles in this Implementation Guide

Description of Profiles, Differentials, Snapshots and how the different presentations work.

NameFlagsCard.TypeDescription & Constraints    Filter:
Bindings
Constraints
Obligations
fr-ligne-prescription 0..* Base Ligne de precription
traitement 0..1 Base Identification du traitement concerné par la ligne de prescription
identifiant 0..1 Identifier Identifiant du traitement prescrit
dosage 0..* Base Quantité de substance active dans une unité de dispensation
valeur 0..1 decimal Valeur du dosage
unite 0..1 code Unité du dosage
forme 0..1 code Forme galénique
dateDebut 0..1 dateTime Date de début de la ligne de prescription
dateFin 0..1 dateTime Date de fin de la ligne de prescription
posologie 0..* Base Structure de la posologie par séquences de prise.
sequence 0..1 decimal Numéro de séquence. La séquence s+1 commence à la fin de la séquence s. En cas de séquences ayant le même numéro, celles-ci se déroulent simultanément.
voieAdministration 0..1 code Voie d'administration du traitement
siteAdministration 0..1 code Région anatomique d'administration du traitement
quantitePrescrite 0..* Base Quantité de traitement prescrite
valeur 0..1 decimal Quantité à prendre par prise
valeurMax 0..1 decimal Quantité maximale à prendre par prise
unite 0..1 code Unité de la quantité prescrite (ex : comprimé, mg, ...)
quantiteMaxParPeriode 0..* Base Quantité maximale par unité de temps
valeurQuantite 0..1 decimal Valeur maximale pour l'unité de temps donnée
uniteQuantite 0..1 code Unité de la quantité
valeurTemps 0..1 decimal Valeur du temps
uniteTemps 0..1 code Unité de temps
evenementDeclenchant 0..* Base Evenement déclenchant de la prise
code 0..1 CodeableConcept Code ou texte de l'évènement déclenchant
offset 0..1 unsignedInt Temps en minute avant/après l'élément déclenchant
condition 0..* CodeableConcept Code ou texte de la condition sous laquelle le traitement doit être pris (ex : en cas de douleurs).
instructionPatient 0..1 string Instruction au patient sous forme textuelle
frequence 0..1 Base Description de fréquence de prise
valeur 0..1 decimal Nombre de prise de la quantité "quantitePrescrite" par période
repetitionPeriode 0..1 code nombre de prise par période (ex : le 3 dans une fois tous les trois jours)
unitePeriode 0..1 code unité de la période (ex : jour dans le 3 dans une fois tous les trois jours)
jourSemaine 0..* code Jour de la semaine de la prise
heurePrise 0..1 time Heure de la prise
precision 0..1 string Instruction additionelle
dureeAdministration 0..1 Base Rythme d'administration
duree 0..1 decimal Durée de l'administration
dureeUnite 0..1 code Unité de la durée d'administration
dureeMax 0..1 decimal Durée maximale de l'administration
dateDebut 0..1 dateTime Date de début de la séquence de traitement
dateFin 0..1 dateTime Date de fin de la séquence de traitement
dureeTraitement 0..1 Base Durée du traitement - la durée du traitement peut être indiquée en complément ou à la place des dates de début et de fin de traitement
valeur 0..1 string Valeur de la durée de traitement
unite 0..1 code Unité de la durée de traitement
